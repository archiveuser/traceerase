# TraceErase

Находит и стирает цифровой след. Вводишь ник, email или домен — движок в реальном
времени опрашивает 59 источников и стримит находки в браузер по SSE.

**Сканирование настоящее.** Никаких моков: живые HTTP-запросы к сайтам, официальные API
Bluesky и Mojang, DNS-over-HTTPS, RDAP (whois), Certificate Transparency логи, Gravatar.
На демонстрации можно вводить свои реальные данные — жюри увидит свои же аккаунты.

## Запуск

```bash
npm start          # → http://localhost:3000
npm test           # проверяет логику определения аккаунтов
```

Нужен Node 18+. Зависимостей нет — только stdlib.

Опционально: `HIBP_KEY=<ключ>` включает проверку утечек через Have I Been Pwned.
Без ключа блок честно пишет, что проверка выключена, а не показывает пустой результат.

## Что проверяется

| Ввод | Источники |
|------|-----------|
| никнейм | 59 сайтов: GitHub, GitLab, Telegram, Steam, Reddit, VK, Bluesky, Minecraft, Habr, Mastodon, YouTube, Tumblr, LiveJournal, Last.fm, Behance и др. |
| email | Gravatar (аватар + публичный профиль), MD5-хеш, MX-записи домена, утечки (с ключом HIBP) |
| домен | A/TXT-записи, RDAP-регистрация, поддомены из CT-логов |


## Принцип, на котором держится доверие

Три состояния, и они не схлопываются:

- `found` — след точно есть
- `free` — точно чисто
- `unknown` — источник не ответил

Ложный `found` дороже десяти пропусков: если жюри увидит «найден аккаунт», которого нет,
доверие к продукту кончится на месте. Поэтому всё сомнительное уходит в `unknown`,
а упавший источник никогда не показывается как «чисто».

Каждый источник в `sites.json` проверен мусорным ником: **ни один не даёт ложный плюс**.
Те, что врали (PyPI, Replit, Kaggle, Spotify, Imgur, Bluesky-веб и ещё десяток), либо
заменены на официальные API, либо выброшены. Те, что всегда молчат из-за бот-стен
(npm, Medium, CodePen, Hacker News…), выброшены тоже: строка «требует проверки»
на каждом скане — это шум, а не информация.

Правила в `scan.js`, которые до этого довели:

- soft-404 (Telegram, Steam, Pinterest, VK) отличаются маркером в теле ответа;
- редирект с профиля = профиля нет, каким бы кодом ни ответили. GitLab шлёт 403 на
  `/users/sign_in`, WordPress и Bandcamp — 200 на служебную страницу. Ник в query
  (`?subdomain=`, `?new_domain=`) за профиль не считается;
- у `has`-сайтов маркер читается даже при 400 (так отвечает API Bluesky на
  несуществующий хендл), но 403/429 остаются `unknown` — это бот-стена, судить не по чему;
- запросы идут пулом по 12. Сто параллельных с одного IP собирают бан вместо ответов.

`npm test` держит все эти правила на месте.

## Дизайн

1-bit: ни одного цветного пикселя, проверено обходом всех вычисленных стилей.
Статус читается формой — заливка / контур / штриховка, а не оттенком. Это работает
для дальтоников и на чёрно-белой печати.

Шрифт — Monocraft (клон майнкрафтовского, OFL), подмножество с кириллицей, 27 КБ,
раздаётся со своего сервера: ни одного запроса к CDN, работает офлайн.
Пиксельный — только для заголовков, цифр и кнопок; текст набран обычным моно,
иначе абзацы нечитаемы.

## Ограничения

- Цифры в блоках «Проблема» и «Результаты» — маркетинговые, не из сканирования.
- Кнопка «Удалить» генерирует готовое GDPR-требование для ручной отправки.
  Автоматическая рассылка — это Pro, и её ещё нет.
- Адрес в письме (`privacy@<домен>`) угадан из хоста, а не взят из базы реальных контактов.
- 18 стеклянных панелей с `backdrop-filter` + SVG-рефракция — тяжело для слабых машин.
  Если тормозит: удалить блок `@supports (backdrop-filter: url(#liquid))` в `style.css`,
  затем снизить `--g-blur`.
