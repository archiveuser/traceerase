# TraceErase

Собирает карту публичного цифрового следа и превращает её в маршрут действий.
Каталог содержит **246 сервисов**: 9 источников с доказуемым автоматическим ответом
и 237 официальных маршрутов для ручной проверки. Результаты стримятся в браузер по SSE.

**Автоматическое сканирование настоящее.** Никаких моков: используются публичные ответы
GitHub, GitLab, Gitea, Telegram, Bluesky, Mojang, Codeforces, Chess.com и Lichess,
а для email/домена — DNS-over-HTTPS, RDAP, Certificate Transparency и Gravatar.
Сервисы с авторизацией, бот-стеной или нестабильным URL не имитируются: TraceErase
открывает их как ручной маршрут и не присваивает им `found/free`.

## Запуск

```bash
npm start          # → http://localhost:3000
npm test           # проверяет логику определения аккаунтов
```

Нужен Node 18+. Зависимостей нет — только stdlib.

Опционально: публичная проверка утечек требует одновременно `HIBP_KEY=<ключ>` и
`ENABLE_PUBLIC_HIBP=true`. Второй флаг намеренно выключен по умолчанию: не превращайте
публичный endpoint в неаутентифицированный прокси к HIBP.

## Принцип, на котором держится доверие

Три состояния, и они не схлопываются:

- `found` — источник вернул публичную страницу/запись с таким идентификатором;
  это **кандидат на совпадение**, а не доказательство личности
- `free` — прямое совпадение не обнаружено именно в момент этой проверки;
  это не обещание «в интернете ничего нет»
- `unknown` — ответ не позволяет сделать вывод (бот-стена, лимит, таймаут, неожиданный формат)

Ложный `found` дороже десяти пропусков. Поэтому всё сомнительное уходит в `unknown`,
а упавший источник никогда не показывается как «не найдено».

Автоматический allowlist намеренно короткий. Остальные сервисы остаются полезны в каталоге,
но работают как прямые официальные ссылки. Например, MAX документирует поиск по номеру,
QR-коду или пригласительной ссылке, но не стабильный `max.ru/<username>`; VK требует
отдельного API-контракта. Оба поэтому честно помечены `manual`.

Контракты сверены с первичными источниками: [GitHub Users API](https://docs.github.com/en/rest/users/users#get-a-user),
[GitLab Users API](https://docs.gitlab.com/api/users/), [Telegram usernames / t.me](https://telegram.org/faq#usernames-and-t-me),
[Bluesky identity resolution](https://docs.bsky.app/docs/advanced-guides/resolving-identities) и
[официальная справка MAX о способах связи](https://help.max.ru/help/messages/kto-mozhet-pisat-mne).

Правила в `scan.js`, которые до этого довели:

- `401/403/429/5xx` всегда становятся `unknown`, а не «не найдено»;
- GitLab классифицируется по точному `username` в JSON, а не по похожей строке;
- soft-404 Telegram определяется маркером; Bluesky и Codeforces явно разрешают
  свой документированный отрицательный `400`;
- редирект считается отрицательным только когда точного сегмента ника больше нет;
- если ни одна проверка профиля не дала однозначного ответа, индекс не рассчитывается:
  отчёт показывает «недостаточно данных», а не вводящий в заблуждение `0/100`;
- запросы идут ограниченным пулом, а схема при старте проверяет 200+, HTTPS, категории,
  уникальные id и корректный placeholder каждого автоматического endpoint.

`npm test` держит все эти правила на месте.

## Экспорт

- **TXT** — полный личный отчёт с UTF-8 BOM: время, цель, счётчики, метод, причина,
  статус и прямой URL каждой строки.
- **Печать / PDF** — многостраничная печатная версия с совпадениями и неопределёнными
  ответами; строки «не найдено» остаются в полном TXT.
- **PNG-карточка** — обезличенный вариант для демонстрации: исходный идентификатор
  маскируется, а прямые URL не попадают в изображение.

## Дизайн

1-bit: ни одного цветного пикселя, проверено обходом всех вычисленных стилей.
Статус читается формой — заливка / контур / штриховка, а не оттенком. Это работает
для дальтоников и на чёрно-белой печати.

Шрифт — Monocraft (клон майнкрафтовского, OFL), подмножество с кириллицей, 27 КБ,
раздаётся со своего сервера: ни одного запроса к CDN, работает офлайн.
Пиксельный — только для заголовков, цифр и кнопок; текст набран обычным моно,
иначе абзацы нечитаемы.

## Ограничения

- Цифры в блоках «Проблема» и «Результаты» — маркетинговые, не из сканирования.
- Кнопка «Удалить» генерирует готовое GDPR-требование для ручной отправки.
  Автоматическая рассылка — это Pro, и её ещё нет.
- Адрес в письме (`privacy@<домен>`) угадан из хоста, а не взят из базы реальных контактов.
- 18 стеклянных панелей с `backdrop-filter` + SVG-рефракция — тяжело для слабых машин.
  Если тормозит: удалить блок `@supports (backdrop-filter: url(#liquid))` в `style.css`,
  затем снизить `--g-blur`.
